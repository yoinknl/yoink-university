Runtimes

How does a language work?

In general

  • Analysis
  • Translation
  • Execution


* these are not the technical terms

Analysis

Is the code valid?


technical term: tokenize, parse

Translation

Turn it into 1s and 0s


technical term: compile

Execution

Run the program

Two types

interpreted

vs

compiled

Three types

interpreted

vs

compiled

vs

intermediate

Interpreted languages

How?

Run the program:

Analysis + translation + execution

Pros & Cons

  • 👍 Fast development
  • 👍 Platform-independent
  • 👎 Slow

Use cases

  • Scripting
  • Prototyping
  • Simple web backend

Languages

  • JavaScript
  • Python
  • R

Compiled languages

How?

Compile the program:

Analysis + translation


Run the program:

Execution


also called: native

Pros & Cons

  • 👍 Compiled programs are fast
  • 👎 Compilation slows development
  • 👎 Programs are platform-dependent

Use cases

  • Operating systems
  • Mobile apps
  • Embedded systems
  • Real-time sytems
  • Command-line tools

Languages

  • C
  • C++
  • Go
  • Rust

Intermediate

How?

Compile the program:

Analysis + translation

(to intermediate code)


Run the program:

Analysis + translation + execution

(of intermediate code)

Pros & Cons

  • 👍 Execution is almost as fast as compiled
  • 👍 Platform-independent
  • 👎 Compilation slows development

Use cases

  • Enterprise systems
  • Large web backends
  • Mobile apps

Languages

  • Java, Kotlin, Scala: JVM bytecode
  • C#, F#: Intermediate Language

Intermediate:
web edition

How?

JavaScript becomes the intermediate language

  • TypeScript
  • Scala.js
  • Kotlin/JS

How?

WebAssembly becomes the intermediate language

  • C
  • C++
  • Go
  • Rust

Boundaries fade

  • Interpreted → intermediate
    • Python (pyc)
  • Interpreted → native
    • Cython
  • Intermediate → native
    • Scala Native
    • Kotlin/Native
  • Intermediate → interpreted
    • Scala as a scripting language
    • JShell

Summary

  • Interpreted languages
    • no compilation
    • start from scratch every time
  • Compiled languages
    • save translation
    • run it many times
  • Intermediate languages
    • best of both worlds
    • also worst of both worlds
